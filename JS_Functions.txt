Accept_Button
-Turns the corresponding page to "Accepted"
-Run "Handle_Switching"

Decline_Button
-Turns the corresponding page to "Accepted"
-Run "Handle_Switching"

Handle_Switching
-Likely going to change a lot depending on data types
-Organizes and keeps track of accepted/declined pages
-Can be outputted at any moment

Populate_Page
-Scans and collects all the inputs
-Creates buttons for each
-Makes them all unmarked as default
-Assigns the appropriate classes to each

Next_Data, Prev_Data
-Switches the visual to the next/prev data
-Switches the written info to that of the next/prev data
